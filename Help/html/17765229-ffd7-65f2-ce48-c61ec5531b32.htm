Remarks

As long as the end of the source has not been reached and maxCount bytes have not yet been read, the buffer will be repeatedly filled with data read from the source, with fillAction being called with the buffer, an offset, and a length as parameters each time the buffer is filled.

If buffer is present, it is used as the buffer. If buffer is omitted or null, a new array is allocated.

If requireFullCount is true, the enumeration will throw an EndOfStreamException if the end of the source has been reached before maxCount bytes have been read. If false, reaching the end of the source simply causes the enumeration to end early.
